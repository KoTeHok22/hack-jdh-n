# Hypothesis Factory

AI-платформа генерации научных гипотез на базе RAG и мульти-стратегической генерации. Создана для сокращения времени исследователей от 2 недель обзора литературы до 5 минут.

## Ключевые возможности

- **Конвейер OCR** — распознавание фото-документов (Mistral / GLM-OCR / PaddleOCR) с автоматическим переключением
- **Гибридный поиск** — семантический (Qdrant) + по ключевым словам (BM25) с Reciprocal Rank Fusion
- **Мульти-стратегическая генерация** — 3 стратегии: аналогии, заполнение пробелов, междисциплинарный перенос
- **Цепочка рассуждений** — интерпретируемые рассуждения LLM, видимые пользователю
- **Многокритериальная оценка** — 4 критерия (новизна, реализуемость, влияние, риск) + составной балл
- **Граф знаний** — интерактивная визуализация связей (Cytoscape.js)
- **Экспорт** — JSON, CSV, XLSX с полной информацией

## Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd hypothesis-factory

# 2. Настроить переменные окружения
cp .env.example .env
# Отредактировать .env — заполнить YANDEX_API_KEY и YANDEX_FOLDER_ID

# 3. Запустить (без GPU, через Mistral/PaddleOCR):
docker compose up --build

# 3а. Или с GPU (GLM-OCR через vLLM):
docker compose --profile gpu up --build
```

После запуска:
- **Фронтенд:** http://localhost:8000
- **Документация API (Swagger):** http://localhost:8000/docs
- **Проверка работоспособности:** http://localhost:8000/health

## Демонстрационный сценарий (что показать жюри)

1. **Загрузить PDF** на странице «Документы» (перетаскивание)
2. **Дождаться индексации** (~1-2 минуты на документ)
3. **Перейти на Панель управления** → заполнить поле проблемы:
   > Повысить извлечение никеля из хвостов флотации на 15-20%
4. **Нажать «Сгенерировать гипотезы»** (~1-2 минуты)
5. **Просмотреть гипотезы** — табличное представление с составным рейтингом
6. **Открыть детальную карточку** — механизм, ссылки, оценки, цепочка рассуждений
7. **«Граф знаний»** — визуализация связей гипотезы ↔ источники
8. **Экспорт в XLSX** — форматированный отчёт с полной информацией

## Как работает система

### 1. Конвейер загрузки данных

```
PDF/DOCX/TXT/Изображения
      ↓
OCR (Mistral/GLM/Paddle) + Текстовый парсер (PyMuPDF/python-docx)
      ↓
Семантическое разбиение (абзацы → 256-512 токенов, 20% перекрытие)
      ↓
Эмбеддинги (Cloud.ru API, OpenAI-совместимый, 1536-мерный)
      ↓
Qdrant (векторы) + BM25 (индекс ключевых слов) + PostgreSQL (метаданные)
```

### 2. Гибридный поиск (Reciprocal Rank Fusion)

```
Запрос: "жаропрочность сплавов XH77TЮР"
      ↓
┌─────────────────────┬─────────────────────┐
│ Семантический поиск │ Поиск по ключевым словам BM25 │
│ (Qdrant, топ-10)    │ (точное совпадение, топ-10)    │
│ Оценка: 0.92, 0.88..│ Оценка: 12.4, 11.2..           │
└─────────────────────┴─────────────────────┘
            ↓ RRF (k=60)
┌─────────────────────────────────────┐
│ Объединённые результаты (топ-10)    │
│ RRF_балл = Σ 1/(k + rank_i)        │
└─────────────────────────────────────┘
```

Зачем гибрид? Чистый семантический поиск пропускает точные термины («XH77TЮР»). Точное совпадение BM25 + семантический контекст = более высокая точность для научных запросов.

### 3. Мульти-стратегическая генерация гипотез

```
Формулировка проблемы (естественный язык)
      ↓
Анализ LLM → {target_kpi, target_delta, domain, constraints}
      ↓
┌─────────────────────────────────────────┐
│ Стратегия 1: АНАЛОГИЯ (40% слотов)      │
│ «Применить известные решения из похожих…»│
│ Пример: магнитная сепарация +71мкм       │
│                                        │
│ Стратегия 2: ЗАПОЛНЕНИЕ ПРОБЕЛОВ (40%)  │
│ «Что ещё не было опробовано…»           │
│ Пример: новые реагенты-собиратели       │
│                                        │
│ Стратегия 3: МЕЖДИСЦИПЛИНАРНАЯ (20%)   │
│ «Адаптировать из смежных отраслей…»    │
│ Пример: полимерные технологии для флотации │
└─────────────────────────────────────────┘
            ↓
Дедупликация (сходство по Джаккарду > 0.75)
      ↓
Многокритериальная оценка (4 метрики, 1-5)
      ↓
Ранжированные гипотезы + трассировки цепочки рассуждений
```

### 4. Система оценки

Каждая гипотеза оценивается по 4 критериям (шкала 1-5):

| Критерий | 1 | 3 | 5 |
|----------|---|---|---|
| **Новизна** | Стандартная практика | Модификация процесса | Прорыв |
| **Реализуемость** | CapEx >18 мес | Плановое обслуживание | Немедленная смена режима |
| **Влияние** | <1% снижение потерь | 3-7% снижение | >15% снижение |
| **Риск** | Обратимый | Требуются лабораторные испытания | Риск остановки производства |

**Составной балл** = 0.25·новизна + 0.25·реализуемость + 0.30·влияние − 0.20·риск

## Провайдеры OCR

| Провайдер | Тип | Требуется GPU | Качество | Скорость | Стоимость |
|-----------|-----|---------------|----------|----------|-----------|
| **Mistral OCR** | Облачный API | Нет | Отличное (лучший для научных) | ~5-15с/стр | Платно |
| **GLM-OCR** | Локальный (vLLM) | Да (4-8ГБ VRAM) | Очень хорошее | ~10-30с/стр | Бесплатно |
| **PaddleOCR** | Локальный (CPU) | Нет | Хорошее (универсальный) | ~5-20с/стр | Бесплатно |

Автоматическое переключение: при выборе «auto» система автоматически определяет доступный провайдер через проверки работоспособности и выбирает лучший: GLM → Paddle → Mistral.

## Устранение неполадок

### Docker Compose не стартует
```bash
# Проверить что порты 8000, 5432, 6333 свободны:
netstat -tulpn | grep -E "8000|5432|6333"

# Проверить Docker:
docker compose logs backend
docker compose logs db
docker compose logs qdrant
```

### Ошибка API эмбеддингов
Убедиться что `EMBEDDING_API_KEY` правильный. Тест:
```bash
curl -X POST http://localhost:8000/health
# Должен вернуть {"status": "ok", "qdrant": true, "postgres": true}
```

### OCR не работает
Проверить доступность провайдеров:
```bash
curl http://localhost:8000/api/v1/documents/ocr-providers
```
Выбрать доступный провайдер и использовать Mistral как резервный (всегда доступен).

### GLM-OCR требует GPU
GLM-OCR работает только с NVIDIA GPU и CUDA 12.x. Если GPU недоступен — запустить без `--profile gpu`, система автоматически переключится на PaddleOCR/Mistral.
